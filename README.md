## 기능 목록

TDD 방식을  사용하여 실패하는 테스트 코드를 작성하고 → 테스트를 통과할 최소한의 기능 구현 → 기능 구현한 코드를 리팩토링 하는 순서로 진행하겠습니다.

- 입력
    - Test Code (red)
    - 최소 기능 구현 (green)
    - 리팩토링
- 쉼표 또는 콜론을 구분자로 가지는 숫자의 합 반환
    - Test Code (red)
    - 최소 기능 구현 (green)
    - 리팩토링
- 커스텀 구분자 지정
    - Test Code (red)
    - 최소 기능 구현 (green)
    - 리팩토링
- 출력
    - Test Code (red)
    - 최소 기능 구현 (green)
    - 리팩토링
- 사용자가 잘못된 값을 입력할 경우 IllegalArgumentException
    - Test Code (red)
    - 최소 기능 구현 (green)
    - 리팩토링
