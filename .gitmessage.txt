### 꼬리 ###
# <footer>

# 1. **Subject line**
#    1. 변경 사항에 대한 간단한 설명.
#    2. 100자를 넘기지 않도록 한다.
#--------------- TYPE ---------------------------------
# feat (feature): 새로운 기능 추가
# fix (bug fix): 버그 수정
# docs (documentation): 문서 관련
# style (formatting, missing semi colons, …): 스타일 변경
# refactor: 코드 리팩토링
# test (when adding missing tests): 테스트 관련 코드
# chore (maintain): 그 외 자잘한 수정
#-------------------------------------------------------
# 2. **Message body**
#    1. 수정 이유와 전후 비교 설명.
#    2. 명령형 현재 시제로 작성한다. (changed X, change O)
#    3. 100자를 넘기면 줄바꿈한다.
# 3. **Message footer**
#    1. 주요 변경사항은 푸터에 변화에 대한 상세설명, 정의, 이전 노트와 함께 명시되어야 한다.
#    2. 전후를 Before : scope: { ~~ } After : scope: { ~~~ } 와 같이 상세하게 명시한다.
#    3. 처리 완료된, 즉 close 된 이슈에 대해서는 `Closes #123, #124` 로 표기한다.
#    3. 100자를 넘기면 줄바꿈한다.
